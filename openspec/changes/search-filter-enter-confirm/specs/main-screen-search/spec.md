## MODIFIED Requirements

### Requirement: 検索モードの終了
検索モード（入力受付状態）を終了する方法は `Esc` と `Enter` の2種類ある。
`Esc` は常にクエリをクリアして全件表示に戻る。
`Enter` はクエリの有無によって挙動が変わる：
- クエリが空の場合、`Esc` と同じく初期状態に戻る。
- クエリが非空の場合、入力モードを終了するが、クエリと絞り込み結果は保持される（確定）。

確定後は検索バーが引き続き表示され、j/k ナビゲーションと通常の Enter による項目選択が使用できる。

#### Scenario: Escで検索を終了する
- **WHEN** 検索モード中に Esc を押す
- **THEN** 検索バーが非表示になり、パネルが全件表示に戻る

#### Scenario: 空クエリでEnterを押すと初期状態に戻る
- **WHEN** 検索バーにクエリが入力されていない状態で Enter を押す
- **THEN** 検索バーが非表示になり、パネルが全件表示に戻る

#### Scenario: 非空クエリでEnterを押すと検索を確定する
- **WHEN** 検索バーにクエリが入力された状態で Enter を押す
- **THEN** 検索入力モードが終了し（カーソルが非表示になる）、クエリと絞り込み結果は保持されたままになる

### Requirement: 検索中のナビゲーション
検索モード中も j/k キーでフィルタリング済みリスト内をナビゲーションできる。
検索確定後（入力モード終了後）も j/k によるナビゲーションと Enter による項目選択が引き続き使用できる。

#### Scenario: フィルタ結果内をj/kで移動する
- **WHEN** 検索クエリで絞り込まれた状態で j または k を押す
- **THEN** 表示中の絞り込み結果内でカーソルが移動する

#### Scenario: 検索確定後にEnterでストリームを開く
- **WHEN** 検索を確定（入力モード終了）した後、ストリームパネルでカーソルが当たっている項目で Enter を押す
- **THEN** 選択中の（元リストの）ログストリームのイベント画面に遷移する

### Requirement: 検索バーの表示
クエリが非空の場合、アクティブパネルの下部（フッターの上）に検索バーを表示する。
入力モード中は `Search: <query><cursor>` の形式でカーソルを表示する。
確定後（入力モード終了後）はカーソルを表示せず `Search: <query>` の形式で表示し続ける。

#### Scenario: 検索バーに入力内容が反映される
- **WHEN** 検索モード中に "api" と入力する
- **THEN** 検索バーに `Search: api` と表示される

#### Scenario: 確定後も検索バーが表示される
- **WHEN** クエリが非空の状態で検索を確定する（Enter を押す）
- **THEN** 検索バーに `Search: <query>` が表示され続ける（カーソルなし）
